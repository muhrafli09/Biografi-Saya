Cara Kerja Website:

  • Input dari Pengguna: Seorang pengguna mengakses website dengan memasukkan alamat URL (Uniform Resource Locator) pada peramban (browser) seperti Google Chrome, Mozilla Firefox, atau Safari.
  • Permintaan DNS: Browser mengirim permintaan ke sistem DNS (Domain Name System) untuk mendapatkan alamat IP yang terkait dengan domain yang dimasukkan pengguna.
  • DNS Resolusi: Sistem DNS meresolusi nama domain ke alamat IP yang sesuai.
  • Permintaan ke Web Server: Browser mengirimkan permintaan HTTP (Hypertext Transfer Protocol) ke server yang memiliki alamat IP tersebut.
  • Proses di Web Server: Web server menerima permintaan dan mengirimkan kembali berkas HTML, CSS, JavaScript, atau sumber daya lain yang diperlukan ke browser pengguna.
  • Rendering di Browser: Browser menerima berkas-berkas tersebut dan merender halaman web sesuai dengan instruksi yang terdapat di berkas HTML, CSS, dan JavaScript.
  • Tampilan di Layar Pengguna: Halaman web akhirnya ditampilkan di layar pengguna.

Hubungan antara Web Server, Domain, dan Hosting:

  • Web Server: Merupakan perangkat keras atau perangkat lunak yang menyimpan dan mengelola berkas-berkas (seperti HTML, CSS, dan JavaScript) serta melayani permintaan dari pengguna. Web server dapat berupa Apache, Nginx, IIS, dan lainnya.
  • Domain: Merupakan alamat yang mudah diingat yang digunakan untuk mengakses situs web. Contohnya, "www.nama-domain.com". Domain memungkinkan pengguna untuk mengakses situs web tanpa perlu mengingat alamat IP yang rumit.
  • DNS Resolusi: Sistem DNS meresolusi nama domain ke alamat IP yang sesuai.
  • Hosting: Merupakan layanan penyimpanan dan pemuatan berkas-berkas situs web ke server sehingga dapat diakses melalui internet. Hosting menyediakan ruang server dan sumber daya yang diperlukan untuk menjalankan situs web. Layanan hosting dapat disediakan oleh penyedia hosting seperti AWS, Bluehost, HostGator, dan sebagainya.

Hubungan ini dapat dijelaskan sebagai berikut:

  • Pengguna mengakses website dengan memasukkan domain pada browser.
  • Permintaan DNS dikirimkan untuk mengonversi nama domain ke alamat IP server.
  • Browser mengirim permintaan HTTP ke server yang memiliki alamat IP tersebut.
  • Hosting menyediakan tempat penyimpanan dan sumber daya untuk menjalankan situs web di server.
  • Domain digunakan sebagai alamat yang mudah diingat untuk mengakses situs web tersebut.
